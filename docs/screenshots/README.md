# Screenshots

These images are referenced by the top-level `README.md`. They aren't generated automatically —
capture them from the running app and drop them in this folder with the exact filenames below.

## How to capture

1. `npm run dev` and click **Install upscaler** if you haven't already.
2. On macOS, **⇧⌘4 then Space** captures a single window cleanly (or **⇧⌘5** for a region).
3. Save each shot here using the filename in the table, as **PNG**.
4. Aim for a consistent window size; ~1320×880 (the default) looks good. Trim OS chrome if you like.

## Shots to capture

| Filename | What to show |
| --- | --- |
| `search.png` | A search (e.g. `lightning bolt` or `set:mh3`) with the result grid filled, the Filters panel open, and a deck started in the right panel. |
| `upscale-compare.png` | A card open in the detail view with the **before/after slider** mid-drag — the original vs upscaled split is the headline shot. Pick a card with lots of fine text/border for maximum effect. |
| `card-detail.png` | The detail view showing the **printings strip** (a card with many printings, e.g. a staple) with per-printing **prices** visible. |
| `deck-export.png` | The deck panel with several cards + the running price total, and the **Export** dialog open showing the page/bleed/cut-guide options. |

## Tips

- Dark theme generally reads better in a README, but a light-theme shot is welcome too.
- For the before/after shot, a high-detail card (full-art or text-heavy) makes the upscaling
  difference obvious.
- Keep file sizes reasonable (these are docs, not prints) — a few hundred KB each is plenty;
  run them through any PNG optimiser if needed.
